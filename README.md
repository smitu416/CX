# VoC 分析ダッシュボード

> 顧客の声（Voice of Customer）を「分析」で終わらせず、**意思決定・行動**へ接続するインフラ。

## 🚀 起動方法

### 方法 A — npm（推奨）

```bash
npm install        # 初回のみ
npm run dev        # http://localhost:3000 で起動
```

### 方法 B — Python（インストール不要）

```bash
npm start          # http://localhost:8080 で起動
```

## 📊 機能一覧

### タブ ① ダッシュボード
- **KPI カード** — 総件数 / 未対応件数 / 最多カテゴリ / トップ製品
- **グラフ × 4** — 内容分類別・感情タグ・相談方法・相談者区分（Chart.js）
- **ヒートマップ** — 上位 10 製品 × 内容分類の件数を色濃度で可視化

### タブ ② 潜在ニーズ分析
- **内容分類 × 感情タグ マトリクス** — セルクリックで該当データをモーダル表示
- **発信地区 × 感情タグ マトリクス** — 上位 8 地区の感情分布
- **WHY ラダー** — 内容分類を選択すると「顕在ニーズ → 機能的原因 → 感情的不足 → 潜在価値」を自動生成
- **価値転換マップ** — 感情タグ別に「顕在不満 / 欠落価値 / 転換アイデア」を提示

### タブ ③ 課題管理
- **課題カード** — 優先度スコア順（影響度 × 頻度）で一覧表示
- **ステータス管理** — 未対応 → 対応中 → 完了 をボタンで切り替え
- **担当部署 / 対応期限** の入力フィールド
- **フィルター** — 内容分類 / 感情タグ / 通称名 / 相談者区分 / ステータスで絞り込み

### タブ ④ データ一覧
- **全カラムテーブル** — 相談内容は「詳細」ボタンでモーダル表示
- **フリーテキスト検索** + ドロップダウンフィルター

## 📥 Excel（XLSX）の読み込み

1. 画面右上の **「XLSX を読み込む」** ボタンをクリック
2. `.xlsx` ファイルを選択

### 対応シート・列名

| 列名 | 内容 |
|------|------|
| NO. | 連番 |
| 案件管理番号 | 管理 ID（例: H20250300001）|
| 内容分類 | 営業 / 製品 / サービス / 企業活動 / 部品 / ＨＴＣ / 他窓口誘導 / その他 |
| 相談方法 | インターネット / フリーダイヤル / コネクテッド会員 / ＨＴＣ会員 |
| 製品区分 | ４輪 / ２輪 / Jet / その他 |
| 通称名 | N-BOX / FREED / CIVIC TYPE-R 等 |
| 製品タイプ | グレード等 |
| 年式 | 数値（例: 2024）|
| 相談内容 | 本文テキスト（`<標題>` タグは自動除去）|
| 相談者区分1 | 潜在客 / 既納客 / その他 / ＨＭ関係 / 官公庁・団体 |
| 相談者区分2 | 詳細区分（空欄可）|
| 発信地区 | 都道府県 |

> **シート名「提案要望」を自動検出**します。存在しない場合は先頭シートを使用します。

## 🤖 自動分析ロジック

### 感情タグ（相談内容テキストから自動判定）

| キーワード | タグ |
|-----------|------|
| 不安・心配・怖い・困る | 不安 |
| 怒・ひどい・最悪・不満・許せない | 怒り |
| がっかり・期待して・残念 | 失望 |
| 嬉しい・ありがとう・感謝・感動 | 喜び |
| 欲しい・お願い・改善・してほしい | 要望 |
| （いずれにも該当しない）| 意見 |

### 潜在価値タグ（内容分類 × 感情から自動付与）

| 条件 | 価値タグ |
|------|---------|
| 製品 × 不安 | 安心価値 |
| 製品 × 怒り | 信頼価値 |
| 製品 × 要望 | 快適価値 |
| 営業 × 要望 | 入手価値 |
| サービス（全感情）| 対応価値 |
| 感情 = 喜び | ブランド価値 |
| その他 | 情報価値 |

### 優先度スコア

```
優先度スコア = 影響度 × 頻度

影響度（内容分類）:  製品・サービス=5 / 営業・部品=4 / 企業活動=3 / その他=2
頻度（1〜5）:       通称名ごとの件数を全体比率で 1〜5 にスケーリング
```

## 📝 開発メモ

- データはページリロードでリセットされます（意図的な仕様）
- サンプルデータ 15 件が初期表示されます（`SAMPLE_DATA` 配列を編集）
- 感情・価値タグのルールは `detectEmotion()` / `detectValue()` 関数で変更可能
